The Appointment Service provides comprehensive methods to manage medical appointments, including creation, retrieval, updates, status transitions, and availability checks.

## Methods

### getAll

Retrieve all appointments with patient and status information.

```typescript
appointments.getAll(): Promise<Appointment[]>
```

**Returns:**
Array of appointments ordered by date and time (ascending).

<ResponseField name="appointments" type="array">
  Each appointment includes:
  
  <ResponseField name="id" type="string">
    Appointment UUID
  </ResponseField>
  
  <ResponseField name="fecha_cita" type="string">
    Appointment date (YYYY-MM-DD)
  </ResponseField>
  
  <ResponseField name="hora_cita" type="string">
    Appointment time (HH:MM)
  </ResponseField>
  
  <ResponseField name="estado" type="number">
    Status ID
  </ResponseField>
  
  <ResponseField name="estado_nombre" type="string">
    Status name (e.g., 'Programada', 'Completada')
  </ResponseField>
  
  <ResponseField name="patients" type="object">
    Patient information (id, Nombre, Paterno, Materno)
  </ResponseField>
  
  <ResponseField name="motivo" type="string">
    Appointment reason
  </ResponseField>
  
  <ResponseField name="urgente" type="boolean">
    Whether the appointment is urgent
  </ResponseField>
  
  <ResponseField name="consultorio" type="number">
    Office/room number
  </ResponseField>
</ResponseField>

**Example:**

```typescript
import { api } from '../lib/api';

// From Calendar.tsx
const appointments = await api.appointments.getAll();
console.log(`Total appointments: ${appointments.length}`);
```

**Cache:**
Results are cached for 20 minutes.

### createSecure

Create a new appointment using the secure RPC function.

```typescript
appointments.createSecure(dto: AppointmentDTO): Promise<Appointment>
```

<ParamField path="dto" type="object" required>
  Appointment data transfer object
  
  <ParamField path="dto.id_paciente" type="string" required>
    Patient UUID
  </ParamField>
  
  <ParamField path="dto.fecha_cita" type="string" required>
    Appointment date (YYYY-MM-DD)
  </ParamField>
  
  <ParamField path="dto.hora_cita" type="string" required>
    Appointment time (HH:MM)
  </ParamField>
  
  <ParamField path="dto.motivo" type="string" required>
    Reason for appointment
  </ParamField>
  
  <ParamField path="dto.consultorio" type="number" required>
    Office/room number
  </ParamField>
  
  <ParamField path="dto.duracion_minutos" type="number" required>
    Duration in minutes
  </ParamField>
  
  <ParamField path="dto.tipo_consulta" type="string" required>
    Consultation type
  </ParamField>
  
  <ParamField path="dto.tiempo_evolucion" type="number">
    Evolution time (numeric value)
  </ParamField>
  
  <ParamField path="dto.unidad_tiempo" type="'horas' | 'dias' | 'semanas' | 'meses'">
    Time unit for evolution
  </ParamField>
  
  <ParamField path="dto.sintomas_asociados" type="string[]">
    Associated symptoms array
  </ParamField>
  
  <ParamField path="dto.urgente" type="boolean">
    Whether appointment is urgent
  </ParamField>
  
  <ParamField path="dto.notas" type="string">
    Additional notes
  </ParamField>
</ParamField>

**Example:**

```typescript
import { api } from '../lib/api';

// From CitasPage.tsx
const createPayload = {
  id_paciente: selectedPatient.id,
  fecha_cita: '2024-03-20',
  hora_cita: '10:00',
  motivo: 'Consulta general',
  consultorio: 1,
  duracion_minutos: 30,
  tipo_consulta: 'Primera vez',
  urgente: false
};

const result = await api.appointments.createSecure(createPayload);
```

**Note:**
This method uses the `agendar_cita` RPC function which automatically retrieves `user_id` and `idbu` from the authenticated session.

**Cache invalidation:**
Clears `'all'`, `patient_${id_paciente}`, and `date_consultorio_${fecha}_${consultorio}` caches.

### update

Update an existing appointment.

```typescript
appointments.update(
  id: string,
  dto: Partial<AppointmentDTO>,
  newStatusId?: number
): Promise<Appointment>
```

<ParamField path="id" type="string" required>
  Appointment UUID
</ParamField>

<ParamField path="dto" type="object" required>
  Partial appointment data to update
</ParamField>

<ParamField path="newStatusId" type="number">
  Optional new status ID (triggers history record)
</ParamField>

**Example:**

```typescript
import { api } from '../lib/api';

// From CitasPage.tsx
const updatePayload = {
  hora_cita: '11:00',
  notas: 'Patient requested time change'
};

await api.appointments.update(
  editingAppointment.id,
  updatePayload,
  2 // New status: Confirmada
);
```

**History tracking:**
If `newStatusId` is provided, a record is inserted into `tcCitasHistorial` tracking the status change.

### checkSlotAvailability

Check if a time slot is available for scheduling.

```typescript
appointments.checkSlotAvailability(
  fecha: string,
  hora_inicio: string,
  duracion_minutos: number,
  consultorio: number
): Promise<{ available: boolean; reason: string }>
```

<ParamField path="fecha" type="string" required>
  Date in YYYY-MM-DD format
</ParamField>

<ParamField path="hora_inicio" type="string" required>
  Start time (HH:MM)
</ParamField>

<ParamField path="duracion_minutos" type="number" required>
  Duration in minutes
</ParamField>

<ParamField path="consultorio" type="number" required>
  Office/room number
</ParamField>

**Example:**

```typescript
const slot = await api.appointments.checkSlotAvailability(
  '2024-03-20',
  '10:00',
  30,
  1
);

if (slot.available) {
  console.log('Slot is available!');
} else {
  console.log(`Not available: ${slot.reason}`);
}
```

**RPC function:**
Uses the `verificar_slot` database function.

### getAllowedStatusTransitions

```typescript
appointments.getAllowedStatusTransitions(
  currentStatusId: number
): Promise<number[]>
```

<ParamField path="currentStatusId" type="number" required>
  Current appointment status ID
</ParamField>

**Returns:**
Array of allowed destination status IDs.

**Example:**

```typescript
const allowedStatuses = await api.appointments.getAllowedStatusTransitions(1);
// Returns: [2, 6, 11] - Can transition to Confirmada, Cancelada, or Urgencia
```

### getFilteredStatusOptions

Get filtered status options based on current status.

```typescript
appointments.getFilteredStatusOptions(
  currentStatusId?: number
): Promise<Status[]>
```

<ParamField path="currentStatusId" type="number">
  Current status ID (undefined for new appointments)
</ParamField>

**Returns:**
Array of status objects that are valid transitions.

**Example:**

```typescript
// For new appointment
const initialStatuses = await api.appointments.getFilteredStatusOptions();
// Returns: Programada (1) and Urgencia (11)

// For existing appointment
const nextStatuses = await api.appointments.getFilteredStatusOptions(1);
// Returns valid transitions from Programada
```

### getAppointmentHistory

Retrieve change history for an appointment.

```typescript
appointments.getAppointmentHistory(
  appointmentId: string
): Promise<HistoryRecord[]>
```

<ParamField path="appointmentId" type="string" required>
  Appointment UUID
</ParamField>

**Returns:**
Array of history records with status changes, ordered by most recent first.

<ResponseField name="history" type="array">
  <ResponseField name="id" type="string">
    History record ID
  </ResponseField>
  
  <ResponseField name="estado_anterior" type="number">

  <ResponseField name="estado_nuevo" type="number">
    New status ID

  <ResponseField name="estado_anterior_nombre" type="string">

  </ResponseField>
  
  <ResponseField name="usuario_nombre" type="string">

  </ResponseField>
  
  <ResponseField name="created_at" type="string">

  </ResponseField>
  
  <ResponseField name="notas" type="string">
